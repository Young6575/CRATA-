# CRATA 노트북/데스크탑 운영 가이드

## 역할

- 데스크탑: 실행 서버 + GPU 영상 작업장
- 노트북: 접속 클라이언트 + 코드/에이전트/스킬 편집기
- GitHub: 코드, 지식, 스킬 동기화
- Tailscale: 외부 접속 통로

## 데스크탑 서버 실행

데스크탑에서 실행합니다.

```powershell
cd "C:\Users\wnsdu\Desktop\프로젝트\CRATA 에이전트_CLAUDE"
.\scripts\start-desktop-server.ps1
```

이 스크립트는 서버를 켜기 전에 `git pull --ff-only`를 자동으로 실행합니다. 추적 중인 파일에 로컬 수정이 있으면 덮어쓰지 않기 위해 pull을 건너뛰고 현재 파일로 서버를 시작합니다.

자동 pull 없이 현재 파일 그대로만 서버를 켜고 싶으면:

```powershell
.\scripts\start-desktop-server.ps1 -SkipPull
```

노트북에서는 Tailscale IP로 접속합니다.

```text
http://<데스크탑-Tailscale-IP>:8765
```

## Git 동기화 원칙

작업 전:

```powershell
git pull
```

데스크탑 서버를 새로 켤 때는 위 명령을 따로 입력하지 않아도 `.\scripts\start-desktop-server.ps1`가 자동으로 pull합니다.

서버가 이미 켜져 있는 동안 노트북에서 코드를 push했다면 대시보드 상단의 `업데이트` 버튼을 누릅니다.

- `dashboard.html`만 바뀐 경우: 업데이트 후 화면 새로고침으로 반영됩니다.
- `server.py`가 바뀐 경우: 업데이트는 받아오지만 실행 중인 Python 서버는 이전 코드로 계속 돌기 때문에 데스크탑 PowerShell에서 `Ctrl+C` 후 `.\scripts\start-desktop-server.ps1`로 다시 켭니다.
- 데스크탑에 추적 중인 로컬 수정이 있으면 자동 업데이트는 중단됩니다. 덮어쓰지 않기 위한 안전장치입니다.

작업 후:

```powershell
git add dashboard.html server.py AGENTS.md CLAUDE.md .agents 지식 결과지문구 rules decisions 검수 insights 기획안 대상별_맥락
git commit -m "update crata agent workspace"
git push
```

## Git에 올리지 않는 것

- 영상 원본 파일
- 렌더링 결과
- 로컬 실행 상태 JSON
- `처리관리/local_settings.json`
- 개인 인증/환경변수 파일

## 운영 기준

- 채팅, 작업 상태, 검수 상태는 데스크탑 서버의 로컬 `처리관리`를 기준으로 봅니다.
- 노트북은 데스크탑 서버에 접속해서 같은 화면을 봅니다.
- 에이전트/스킬/지식 파일을 수정하면 GitHub에 push하고, 반대쪽 PC는 작업 전 pull합니다.
