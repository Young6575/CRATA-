# revise-phrase — 결과지 문구 수정

기존 MASTER 문구의 ` ```text``` ` 블록 본문을 다듬는다. **연결키·결과지 위치는 절대 바꾸지 않는다.**

## 워크플로

1. **대상 식별** — 연결키 또는 결과지 위치로 `크라타 검사 지식 파일/*MASTER.md`에서 해당 문구를 찾는다.
2. **컨텍스트 로드 (필수, 작업 전)**
   - 해당 유형의 지식 정의 → `지식/<검사>/`
   - 같은 결과지 위치의 다른 변주(나이대/유형) 문구 — 톤·구조 비교용
   - 관련 과거 근거 → `decisions/log/` 에서 같은 유형·위치 엔트리
   - 작법 → `rules/phrase_format.md` + 검사별 작법(`rules/org_writing.md` 등)
3. **정합성 점검** — 사용자 요청이 지식 정의와 애매·대립하면 **멈추고 짚는다**(CLAUDE.md 상시 감시). 필요하면 `revise-knowledge`로 분기.
4. **초안 제안** — 변경 전 → 후를 나란히 보여준다. 본문만. 아직 파일 수정 안 함.
5. **확인** — 사용자 OK 후에만 반영.
6. **반영 + 근거 기록**
   - MASTER의 `text` 블록 본문만 Edit.
   - `decisions/log/YYYY-MM-DD_제목.md` 작성 (`층: 문구`). 형식은 `decisions/README.md`.

## 체크리스트
- [ ] 연결키·결과지 위치 그대로 두었는가
- [ ] 지식 정의와 모순 없는가
- [ ] 같은 위치 다른 변주와 톤이 어긋나지 않는가
- [ ] 근거 엔트리를 남겼는가
