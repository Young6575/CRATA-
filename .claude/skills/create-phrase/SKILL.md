# create-phrase — 새 문구 생성

비어 있는 좌표(검사/페이지/박스/유형, 개인검사는 나이대 포함)에 새 문구를 만든다.

## 워크플로

1. **좌표 확정** — 어떤 검사, 어느 페이지/박스, 어떤 유형(개인검사면 나이대)인지 확정한다.
2. **컨텍스트 로드 (필수, 작업 전)**
   - 그 위치의 의미 → MASTER 파일 상단의 페이지/박스 위치 기준표
   - 대상 유형의 지식 정의 → `지식/<검사>/`
   - **같은 위치의 다른 변주** 문구 — 톤·길이·구조의 본보기
   - 작법 → `rules/phrase_format.md` + 검사별 작법
   - 관련 과거 근거 → `decisions/log/`
3. **연결키 부여** — `rules/phrase_format.md`의 네이밍 규칙대로. **파일 전체에서 중복 없는지 확인.**
4. **정합성 점검** — 지식 정의와 어긋나면 멈추고 짚는다.
5. **초안 제안** — 연결키·결과지 위치·동일 위치 설명·`text` 본문을 갖춘 완전한 블록으로 제시. 아직 파일에 안 넣음.
6. **확인** — 사용자 OK 후에만 반영.
7. **반영 + 근거 기록** — MASTER의 알맞은 위치에 삽입, `decisions/log/` 엔트리(`층: 문구`).

## 체크리스트
- [ ] 연결키가 규칙에 맞고 유일한가
- [ ] 같은 위치 다른 변주와 톤·구조가 어울리는가
- [ ] 지식 정의와 모순 없는가
- [ ] 근거 엔트리를 남겼는가
