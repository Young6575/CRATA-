# promote-rule — 근거 → 규칙 승격 루프

`decisions/log/`에 누적된 `## 도출 원칙`을 모아 **같은 뿌리의 원칙이 2회 이상 반복**되면 `rules/`로 승격 후보를 만든다. 이 에이전트가 시간이 지나며 *학습*하는 닫힌 루프의 닫는 쪽 손이다.

## 절대 원칙

1. **자동 반영 금지.** 후보 → 사용자 컨펌 → 반영. 룰은 한 번 박히면 모든 작업에 영향이므로 반드시 짚는다.
2. **근거 링크 필수.** 승격된 룰에는 어느 decisions 엔트리에서 왔는지 링크를 단다(추적 가능성).
3. **이미 있는 룰과 중복 금지.** rules/ 4개 파일을 먼저 다 읽고, 겹치면 강화·통합으로 처리.
4. **표현이 달라도 의미가 같으면 한 군집.** 단순 키워드 매칭 X, 의미 군집.

## 워크플로

### 1단계 — 수집
- `decisions/log/*.md` 전부 읽기.
- 각 엔트리에서 `## 도출 원칙` 섹션만 추출. 없는 엔트리는 건너뜀.
- 엔트리별 메타(날짜·층·검사·대상)도 같이 들고 있는다.

### 2단계 — 의미군집
- 추출된 원칙들을 의미적으로 묶는다. 표현이 달라도 같은 뿌리면 한 군집.
- 예: "지식에 없는 용어 안 끌어들이기" + "한 개념 정의는 한 파일에만" → "단일 출처" 군집.
- 각 군집의 **공통 뿌리 한 문장**을 뽑는다.

### 3단계 — 기존 rules 대조
- `rules/` 전체(`korean_style.md`, `org_writing.md`, `phrase_format.md`, `crata_expression.md`) 먼저 읽기.
- 군집의 뿌리가 이미 있으면 → **강화 후보**(예시·반례 추가)
- 없으면 → **신규 룰 후보**

### 4단계 — 분류
신규 룰 후보는 어느 파일에 들어갈지 판정:
| rules 파일 | 들어가는 것 |
|---|---|
| `korean_style.md` | 한국어 자연스러움·어휘·동사 사용 규약 |
| `phrase_format.md` | 결과지 문구 형식·구조·연결키·블록 |
| `org_writing.md` | 조직검사 문구 작법 (나이대 없음, 유형·축 변주) |
| `crata_expression.md` | CRATA 전반 표현 톤·금기·낙인 가드 |
| (해당 없음) | 새 rules 파일이 필요한지 사용자에게 질문 |

### 5단계 — 리포트
각 군집마다 아래 형식으로 한 묶음:

```
[군집 N] {공통 뿌리 한 문장}

근거 엔트리 (X회):
  - decisions/log/YYYY-MM-DD_xxx.md — "원본 표현 발췌"
  - decisions/log/YYYY-MM-DD_yyy.md — "원본 표현 발췌"

대조:
  - 기존 rules에 있음: rules/zzz.md "기존 표현" → 강화 제안
  또는
  - 기존 rules에 없음 → 신규 후보 (대상 파일: rules/aaa.md)

승격 초안:
  > {rules 파일에 추가할 룰 문장 — 일반화된 형태}
  > 근거: [엔트리1](../decisions/log/...), [엔트리2](../decisions/log/...)
```

### 6단계 — 컨펌
- 군집별로 사용자에게 **개별 컨펌**. 한 묶음으로 묻지 않는다(단계별 컨펌 원칙).
- 사용자가 "OK"한 군집만 → 해당 rules 파일에 Edit으로 추가.
- 보류·기각된 군집은 다음 호출 때 다시 후보로 올라오므로 별도 처리 불필요.

### 7단계 — 사후
- 승격 완료 후 decisions 엔트리는 **그대로 둔다**(역사 보존). 룰의 근거 링크가 가리키므로 충분.
- CLAUDE.md의 어느 원칙에 해당하는지 명시되지 않은 군집이 나오면 사용자에게 짚어준다(CLAUDE.md 갱신 필요할 수 있음).

## 호출 빈도

- 새 decisions 엔트리 5개+ 누적될 때
- 사용자가 명시적으로 호출할 때
- 같은 종류 교정을 반복받고 있다고 느낄 때 (에이전트 자발 제안)

## 주의

- decisions 엔트리에 `## 도출 원칙`이 없는 경우(plaud 로그 등 단순 기록)는 군집 대상이 아니다.
- 1회만 등장한 원칙은 **승격하지 않는다.** "반복"이 핵심 신호.
- 단, 한 엔트리 안에서 사용자가 *명시적으로* "이건 일반 규칙이다"라고 말한 경우는 예외 — 별도로 짚는다.
