# audit-phrases — 정합성 점검

문구를 **고치지 않고** 문제를 찾아 리포트한다. 실제 수정은 `revise-phrase`로 넘긴다.

## 워크플로

1. **범위 정하기** — 한 페이지 / 한 유형 / 한 검사 전체 중 무엇을 볼지 정한다.
2. **기계적 점검** (가능하면 hook 검증 스크립트 결과 활용)
   - 연결키 **중복** 없는가
   - 빈 ` ```text``` ` 블록 없는가
   - 구조(연결키/결과지 위치/text) 깨진 항목 없는가
3. **누락 점검** — 같은 위치에 있어야 할 변주가 빠지지 않았는가 (개인검사: 나이대 5종 / 조직검사: 해당 유형·축).
4. **개념 정합성** — 각 문구가 `지식/<검사>/`의 유형 정의와 모순되지 않는가.
5. **톤 일관성** — 같은 위치의 변주들 사이 톤·길이·구조가 들쭉날쭉하지 않은가.
6. **리포트** — 발견 항목을 우선순위로 정리(심각: 중복·빈블록·모순 / 보통: 누락 / 약함: 톤). 각 항목에 연결키와 권장 조치를 단다.

## 주의
- 이 스킬은 **읽기·진단 전용.** 파일을 직접 수정하지 않는다.
- 큰 MASTER 파일을 다 읽어야 하면 컨텍스트 보호를 위해 범위를 좁히거나 Explore 서브에이전트 활용을 고려한다.
