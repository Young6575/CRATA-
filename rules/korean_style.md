# 한국어 문구 작법 — AI 티 제거

결과지 text 블록을 쓸 때 적용하는 한국어 문체 규칙. 공통 포맷은 `phrase_format.md`, 검사별 작법은 `org_writing.md` 참조.

> 출처: 한글 AI 티 분류는 `Humanize KR`(github.com/epoko77-ai/im-not-ai)의 taxonomy에서 CRATA 문구에 맞는 항목만 추렸다. 전체 하네스는 쓰지 않고 체크리스트만 차용.

CRATA 문구는 **짧은 제약 카피**다. 아래는 길이를 늘리지 않고 문체만 다듬는 기준이다.

## 1. 번역투 제거
- "~를 **통해**" → "~로", "~으로"
- "~에 **대해**서" → "~을", "~에 관한"
- "~에 **있어서**" → "~에서", "~할 때"
- 이중 피동 "~되어진다 / 만들어지게 된다" → "~된다 / 만든다"
- "~을 **가지고 있다**" → "~이 있다"

## 2. AI 관용구 금지
- "결론적으로", "시사하는 바가 크다", "주목할 만하다", "혁신적인", "~라고 할 수 있다"
- 이런 상투구는 문구에서 통째로 뺀다. 단정이 필요하면 평이한 서술로.

## 3. 과잉 수식 절제
- "매우", "정말", "굉장히", "너무" 등 강조 부사는 기본 삭제.
- "~적 / ~성 / ~화"의 한자어 추상 접미사 남발 주의 (예: "효율적 측면에서의 강점" → "효율이 강점").
- 동의어 이중 수식 금지 ("명확하고 분명한" → 하나만).

## 4. Hedging(완곡) 남용 주의
- "~할 수 있을 것으로 보인다", "~인 경향이 있다고 볼 수 있다" 같은 다중 완곡은 한 겹으로.
- 단, `org_writing.md`의 "단정·낙인 금지"와 충돌하지 않게 — **불확실해서 흐리는 완곡**은 빼되, **환경 적합성을 여지로 두는 표현**("~한 환경에서 강점이 살아난다")은 유지.

## 5. 형식명사 군더더기
- "~하는 **것이다**" → "~한다"
- "중요한 **점은**", "필요한 **부분이**" → "중요한 건", "필요한 건" 또는 직접 서술
- "~할 **필요가 있다**" → "~해야 한다 / ~하면 좋다"

## 적용 원칙
- 의미는 한 글자도 바꾸지 않는다 (지식 정의·연결키·결과지 위치 불변).
- 탐지된 표현에만 손댄다. 멀쩡한 문장을 새로 쓰지 않는다(과윤문 금지).
- 다듬은 근거가 한 패턴이라도 `decisions/`에 남긴다.
