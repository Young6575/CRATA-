# 문구 포맷 규칙 (공통)

모든 결과지 MASTER 파일(`크라타 검사 지식 파일/*MASTER.md`)이 공유하는 문구 구조 규칙. 검사 종류와 무관하게 적용된다.

## 1. 한 문구의 구조

```
#### (문구 제목)
연결키: `P02_유아_IG_해석설명`
결과지 위치: `2페이지 / 박스 2`
동일 위치 설명: (이 위치가 화면에서 무엇인지)

​```text
(화면에 그대로 나가는 본문)
​```
```

- **연결키**: 코드 연결용 식별자. 화면에 안 보임.
- **결과지 위치**: 페이지/박스. 같은 위치 = 화면의 같은 자리.
- **동일 위치 설명**: 그 자리가 무슨 박스인지.
- **text 블록**: 화면에 실제로 나가는 글.

## 2. 절대 규칙

1. **수정은 ` ```text``` ` 블록 본문만.** 연결키·결과지 위치·동일 위치 설명은 건드리지 않는다.
2. **연결키는 불변.** 새 문구를 만들 때만 새 연결키를 부여하고, 기존 연결키는 절대 바꾸거나 재사용하지 않는다.
3. **연결키는 유일해야 한다.** 파일 전체에서 중복 금지.
4. **text 블록은 화면 표시 그대로다.** 마크다운 기호(`#`, `*`, `-` 등)를 장식으로 쓰지 않는다. 줄바꿈도 화면에 반영되므로 의도한 대로만 넣는다.

## 3. 연결키 네이밍

- **개인행동검사**: `P{페이지2자리}_{나이대}_{유형코드}_{항목}`
  - 예: `P02_유아_IG_해석설명`, `P01_성인_소개문`, `P02_공통_화면제목`
  - 나이대: `유아` `아동` `초등고학년중학생` `청소년` `성인` `공통`
  - 유형코드: 동기위치(I/E) + 동기성향(G/D/B/H/A 등) 조합 — 기존 파일의 표기를 따른다.
- **조직검사**: `ORG{페이지}_{유형 또는 공통}_{항목}`
  - 예: `ORG01_공통_결과지제목`, `ORG02_공통_조직업무방식소개`
- `공통` = 나이대·유형과 무관하게 같은 위치에 같은 문구가 나가는 고정 문구.

## 4. 같은 위치 / 다른 변주

- 같은 `결과지 위치`(페이지/박스)는 화면의 같은 자리다.
- 그 자리의 본문은 **변주축**에 따라 달라진다:
  - 개인행동검사 → **나이대 × 유형**
  - 조직검사 → **유형 · 축 · 성장구조** (나이대 없음)
- 문구를 수정·생성할 때는 **같은 위치의 다른 변주들을 함께 보고** 톤·구조의 일관성을 맞춘다.
