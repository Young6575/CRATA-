# 조합 설명

조직검사 조합(256 상태조합, 16 행동유형)의 **개별 상세 설명**을 보관한다. 산출 원리는 `../조합원리.md` 참조.

## 작성 규칙

- 한 조합 = 한 파일. 파일명은 식별 가능한 코드로 (예: `16_OA_책임실행형.md`, `256_O관계_P없음_A전술_S직관.md`).
- 각 설명에는 **상태**를 표시한다: `초안`(자동/임시) / `검토중` / `확정`.
- 조합 설명을 쓸 때는 **구성 축 세부유형의 정의를 먼저 온전히 이해**한 뒤, 결합 시 어떤 모습인지 써낸다. 근거가 된 세부유형 정의를 본문에 명시한다.
- `초안` → `확정`으로 바꿀 때는 **반드시 `decisions/`에 근거를 남긴다.**
- 화면 문구가 아니라 *지식*이다. 결과지 문구는 `크라타 검사 지식 파일/*MASTER.md`에 있다.

## 설명 항목 (권장 틀)

```
## (조합 이름)
- 상태: 초안 / 검토중 / 확정
- 구성: (어떤 축의 어떤 세부유형들이 결합됐는가)
- 정의: (이 조합이 어떤 사람인가)
- 결합 시 모습: (축들이 합쳐졌을 때 실제로 어떻게 작동하는가)
- 강점:
- 주의점:
- AI핵심: (이 조합을 다룰 때 헷갈리지 말아야 할 한 줄)
- 근거: (이 설명의 바탕이 된 세부유형 정의 / 관련 decisions 엔트리)
```
