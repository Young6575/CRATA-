# 조합 원리

조직검사의 결과 유형은 **개별 유형이 따로 주어지는 것이 아니라, 행동방식 4축의 조합으로 산출**된다. 이 문서는 그 산출 원리를 정의한다. (개별 유형 정의는 `유형_16행동유형.md`, `유형_조직10유형.md` 참조)

## 1. 행동방식 4축

| 축 | 약자 | 세부유형(첫 번째 / 두 번째) |
|---|---|---|
| 목적성취방식 | O | 과제형 / 관계형 |
| 능력표현방식 | P | 설계형 / 기술형 |
| 목표실행방식 | A | 전술형 / 전략형 |
| 정보처리방식 | S | 직관형 / 경험형 |

> **자기효능감향상방식(경쟁형·비교형)은 4축에 포함하지 않는다.** 역량 자리에는 들어갈 수 있으나 4축 조합 산출에는 빠진다.
> 표기 노트: "자기효능감향상방식" 이름은 **효능감 표기로 복귀 확정** (2026-05-27 후속 녹음 번복). 하위 경쟁형·비교형 확정. 자세한 정의는 자기효능감검사(잠정명) 지식 디렉토리 신설 후 이전 예정.

## 2. 각 축의 4가지 상태

각 축은 두 세부유형의 존재 여부에 따라 4가지 상태를 가진다.

1. 첫 번째 유형만 있음
2. 두 번째 유형만 있음
3. 둘 다 있음
4. 둘 다 없음

→ 전체 상태 조합 = 4 × 4 × 4 × 4 = **256가지.** 이것이 256 상태조합이다.

## 3. 16 행동유형 산출 (존재패턴)

16 행동유형은 256보다 거친 단위로, **각 축에 값이 하나라도 있는가(count > 0)** 만 보는 존재패턴이다.

- 각 축을 1비트로 표현 → 4비트 → 2^4 = **16가지.**
- 비트 순서는 **O P A S** (목적 · 능력표현 · 목표실행 · 정보처리).
- 예: `1010` = O 있음, P 없음, A 있음, S 없음 → OA 책임실행형.
- 강도형이 아니라 **존재 여부**로만 산출한다는 점이 핵심.

## 4. 256 상태조합과 16유형의 관계

- 256 상태조합 = 각 축의 *세부 상태*(첫째/둘째/둘다/없음)까지 구분한 정밀 단위.
- 16 행동유형 = 256을 *존재 여부*로 뭉갠 거친 단위.
- 따라서 같은 16유형 안에 여러 256 상태조합이 들어갈 수 있다.

## 5. 작성 방침

- 256 상태조합별 상세 설명은 **자동생성 초안에서 시작해 점진적으로 고도화**한다.
- 조합 설명을 쓸 때는 **구성 축 세부유형의 정의를 먼저 온전히 이해**한 뒤, 그것들이 결합되면 어떤 모습인지 써낸다. (`유형_조직10유형.md`의 해당 세부유형 정의를 근거로 삼는다)
- 확정 전까지 상태는 `초안`으로 두고, 확정 시 `decisions/`에 근거를 남긴 뒤 `확정`으로 바꾼다.
- 개별 조합 설명은 `조합설명/`에 둔다.
