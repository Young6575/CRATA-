# revise-knowledge — 지식 수정·고도화

`지식/` 의 단일 출처 markdown을 수정하거나 새 설명을 저작한다. 문구 수정과 같지만 **파급 점검** 단계가 추가된다(지식이 바뀌면 그 정의를 참조하는 문구·다른 지식이 함께 어긋날 수 있다).

## 워크플로

1. **대상 식별** — `지식/<검사>/` 의 해당 파일·섹션.
2. **컨텍스트 로드 (필수)**
   - 현재 정의 전문
   - 관련 과거 근거 → `decisions/log/`
   - **이 정의를 참조하는 곳**: 같은 검사의 다른 지식 문서, 그리고 이 유형·개념을 쓰는 MASTER 문구
3. **근거 확정** — 왜 바꾸는지 명확히 한다. 근거 없는 수정 금지. 사용자 말과 기존 정의가 대립하면 그 지점을 먼저 정리.
4. **파급 점검** — 이 변경이 영향을 주는 다른 지식·문구를 **목록화**한다. 함께 고쳐야 할 것을 빠짐없이 식별.
5. **초안 제안** — 변경 전 → 후 + **파급 목록**을 함께 제시. 아직 파일 수정 안 함.
6. **확인** — 사용자 OK 후에만 반영.
7. **반영 + 근거 기록**
   - 지식 파일 수정. 초안→확정 시 상태(frontmatter/항목)도 갱신.
   - 파급 목록의 다른 문서도 동기화(또는 후속 작업으로 명시).
   - `decisions/log/` 엔트리 작성 (`층: 지식`, **파급 칸 반드시 기재**).

## 조합 설명을 새로 쓸 때
- 먼저 **구성 축 세부유형 정의**(`유형_조직10유형.md` 등)를 온전히 읽는다.
- 결합 시 어떤 모습인지 써내고, 근거가 된 정의를 본문에 명시한다.
- 확정 전까지 `상태: 초안`.

## 체크리스트
- [ ] 근거가 분명한가
- [ ] 파급(영향받는 문구·지식)을 모두 찾았는가
- [ ] 초안/확정 상태를 올바르게 표시했는가
- [ ] decisions 엔트리에 파급을 적었는가
