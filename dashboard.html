CRATA 운영 가이드

회의록, 업무, 일정, 문구 수정을 한 흐름으로 관리합니다.

이 페이지는 대시보드를 처음 열었을 때 어떤 순서로 움직여야 하는지 정리한 내부 운영 문서입니다. 실제 작업은 각 페이지에서 처리하고, 여기서는 흐름과 기준을 빠르게 확인합니다.

회의록 처리PLAUD 전사록을 읽고 업무 단위로 나눕니다.
작업 관리요청 묶음과 세부 업무 진행률을 확인합니다.
일정 등록업무에서 날짜 단서를 뽑아 캘린더 초안을 만듭니다.
기본 업무 흐름
회의록에서 나온 일을 작업과 일정까지 연결하는 표준 순서입니다.
01
PLAUD 업데이트회의록 페이지에서 목록을 갱신하고 미처리 녹음을 확인합니다.
02
전사록 처리업무로 나누기를 실행해 회의 내용을 세부 업무로 분해합니다.
03
작업 묶음 확인작업지시서에서 상위 요청 카드와 세부 업무 상태를 확인합니다.
04
일정 초안 생성날짜나 마감 단서가 있는 업무는 캘린더 채팅에서 일정 초안을 만듭니다.
05
검토와 승인문구·지식 수정안은 채팅에서 검토하고, 승인된 반영만 decisions 기록으로 남깁니다.
작업 스킬 카탈로그
어떤 요청을 어느 에이전트 루틴으로 보내야 하는지 확인합니다.
revise-phrase기존 문구 수정결과지문구 MASTER의 text 블록 본문만 수정할 때 사용합니다.
create-phrase새 문구 작성비어 있는 결과지 위치나 새 항목 문구를 생성합니다.
revise-knowledge검사 지식 수정유형 정의, 조합 원리, 검사 체계 자체를 고도화합니다.
audit-phrases정합성 점검누락, 중복, 지식 정의와의 충돌, 유형 표기 문제를 찾습니다.
export-review대량 검수 문서 생성변경사항이 많아 채팅 검토만으로 부족할 때 docx 파일로 묶습니다.
apply-review승인 후 반영오케이된 피드백만 지식·문구 파일에 반영합니다.
운영 원칙
문구 수정은 본문만연결키와 결과지 위치는 바꾸지 않습니다.
승인 없는 반영 금지제안·피드백은 채팅에서 반복 검토하고, 오케이된 반영만 decisions/log에 기록합니다.
미확정 지식은 단정하지 않기초안 상태의 개념은 결과지 문구 기준으로 쓰지 않습니다.
Claude와 Codex 선택작업 성격에 맞춰 엔진을 고르고 실패 시 다른 엔진으로 재시도합니다.
빠른 실행 루틴
/meeting회의록에서 업무 만들기회의록 페이지에서 미처리 파일을 선택한 뒤 업무로 나누기를 실행합니다.
/task요청을 업무로 정리작업 요청 페이지에서 큰 요청을 입력하고 세부 업무 계획을 확인합니다.
/schedule업무를 일정으로 전환캘린더 채팅에 일정 요청을 입력하고 초안을 확인 후 저장합니다.
/review검수 사이클 관리검수 현황에서 export, 피드백, apply 상태를 추적합니다.

프로젝트 생성 판단

1
업무가 3개 이상으로 분해됨세부 업무가 여러 단계로 나뉘면 프로젝트 후보입니다.
2

3
검수·일정·회의록까지 이어짐PLAUD, 검수, 캘린더가 연결되면 흐름 추적이 필요합니다.
4
한 번에 끝나는 요청은 제외짧은 수정, 단일 문구 변경, 확인성 요청은 작업 지시서에서 바로 처리합니다.
