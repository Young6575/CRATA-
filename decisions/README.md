# 수정 근거 로그 (decisions)

이 폴더는 **"문구·지식을 왜 그렇게 고쳤는가"의 근거를 누적**하는 곳이다. 이 에이전트가 시간이 지나며 *학습*하는 핵심 장치다.

## 왜 남기는가

- 같은 판단을 반복하지 않기 위해. 과거에 내린 결정과 어긋나지 않게.
- 새 문구·지식을 만들 때 과거 근거를 참고해 일관성을 유지하기 위해.
- 반복되는 근거는 `rules/`로 **승격**한다(개별 사례 → 일반 규칙).

## 언제 쓰는가 (필수)

- **문구**를 수정·생성할 때 (`revise-phrase`, `create-phrase`)
- **지식**을 수정·고도화할 때 (`revise-knowledge`)
- 초안 상태 개념을 `확정`으로 바꿀 때

→ **근거 없는 수정은 금지.** 수정과 근거 기록은 한 묶음이다.

## 어떻게 쓰는가

1. `log/` 아래에 `YYYY-MM-DD_짧은제목.md` 파일을 만든다. (한 수정 = 한 파일, 또는 한 세션의 연관 수정 묶음)
2. 아래 형식을 채운다.
3. 작업 전에는 항상 `log/`에서 **관련 대상(같은 유형·위치·개념)의 과거 엔트리를 먼저 읽는다.**

## 엔트리 형식

```
---
날짜: 2026-05-22
층: 문구            # 문구 | 지식
검사: 조직행동
대상: ORG06_RA_강점  # 연결키 / 또는 지식 경로(지식/조직검사/유형_조직10유형.md#실행자형)
상태: 확정          # 제안 | 확정
---

## 무엇을
(변경 전 → 변경 후. 핵심만)

## 왜
(이렇게 바꾼 근거. 어떤 정의·대화·모순 때문인지 구체적으로)

## 파급        # 지식 수정일 때 특히 중요
(이 변경이 영향을 주는 다른 문구/지식. 함께 고쳐야 할 것)

## 도출 원칙    # 있으면
(이 사례에서 끌어낼 수 있는 일반 규칙. 반복되면 rules로 승격)
```

## 승격 (decisions → rules)

같은 `도출 원칙`이 여러 엔트리에서 반복되면, `rules/`의 해당 작법 문서에 규칙으로 추가한다. 추가 시 근거가 된 엔트리들을 링크한다.
